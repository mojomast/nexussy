# Minimal Design Context Pack

## Visual Theme
Quiet, content-first, and distraction-free. Every visual element should justify its presence and support comprehension.

## Color Palette
- Primary: black `#111111` or a single restrained brand accent.
- Neutrals: white, off-white, charcoal, and a small gray scale for borders and secondary text.
- Support: semantic status colors only when actionably useful.

## Typography
Use a simple system sans-serif stack with excellent line length, generous leading, and clear heading levels. Avoid ornamental type.

## Components
Buttons, inputs, cards, and navigation should be plain, predictable, and consistent. Prefer labels over icons when ambiguity is possible.

## Layout
Use a strong grid, large whitespace, linear reading order, and simple page structure. Group related content with spacing before decoration.

## Elevation
Use minimal elevation: borders and whitespace first, light shadows only for overlays or modal focus.

## Do / Don't
- Do remove redundant copy, controls, and decoration.
- Do prioritize accessibility, clarity, and performance.
- Don't add gradients, excessive animation, or complex chrome.
- Don't hide essential context behind clever interactions.

## Responsive Behavior
Stack content naturally, preserve readable measure, keep navigation simple, and ensure all controls remain reachable without horizontal scrolling.

## Agent Prompt Guide
When designing screens, choose the simplest effective solution, document why each element exists, and make empty/error/loading states clear.
