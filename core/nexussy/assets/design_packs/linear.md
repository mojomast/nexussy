# Linear Design Context Pack

## Visual Theme
Focused, fast, keyboard-friendly, and calm. Interfaces should feel like a precise workspace for high-performing product teams.

## Color Palette
- Primary: near-black `#08090A`, graphite `#1C1D21`, and violet/blue focus accents.
- Neutrals: zinc grays, low-contrast dividers, and muted background panels.
- Support: restrained green, yellow, and red status colors only where state matters.

## Typography
Use a compact modern sans-serif stack. Prioritize dense but readable lists, medium-weight headings, and monospaced snippets for IDs, shortcuts, or logs.

## Components
Use command palettes, issue-like rows, badges, tabs, sidebars, and compact toolbars. Components should expose shortcuts and reduce visual noise.

## Layout
Favor split panes, sticky navigation, searchable lists, and detail drawers. Keep information architecture shallow and workflows direct.

## Elevation
Prefer thin borders, translucent panels, and minimal shadows. Depth should be quiet and functional.

## Do / Don't
- Do optimize for speed, scanability, and keyboard operation.
- Do show state, ownership, priority, and timestamps clearly.
- Don't use decorative illustration or marketing-heavy copy inside the app.
- Don't sacrifice accessibility for density.

## Responsive Behavior
Convert split panes to drill-down navigation on small screens, keep search/action bars available, and retain readable row spacing.

## Agent Prompt Guide
When designing screens, emphasize operational clarity, command-driven flows, compact hierarchy, keyboard accessibility, and low-friction team collaboration.
